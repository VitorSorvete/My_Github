    Comandos Git Comuns

    git init → Inicializa um repositório vazio

    git clone → Clona repositório remoto

    git status → Mostra estado da árvore de trabalho

    git add → Adiciona mudanças ao stage

    git commit -m "msg" → Registra snapshot

    git push → Envia commits ao GitHub

    git pull → Sincroniza e integra mudanças

    git branch → Lista ou cria branches

    git merge → Mescla branches

    git log → Histórico de commits

    Resolvendo conflitos

  • 1. Identifique o conflito: Use git status para ver quais arquivos estão em conflito.
  • 2. Edite os arquivos: Abra os arquivos em conflito e procure por marcas de conflito (<<<<<<<, =======, >>>>>>>). Edite o conteúdo para resolver o conflito.
  • 3. Adicione os arquivos resolvidos: Após resolver os conflitos, use git add <arquivo> para marcar os arquivos como resolvidos.
  • 4. Faça o commit: Use git commit para criar um novo commit que inclui as resoluções de conflito.
  • 5. Continue o processo: Se você estava no meio de um rebase ou merge, continue o processo com git rebase --continue ou git merge --continue.
  • 6. Verifique: Use git status novamente para garantir que todos os conflitos foram resolvidos e que o repositório está em um estado limpo.
