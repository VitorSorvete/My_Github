Git é um sistema de controle de versão que permite registrar e gerenciar o histórico de alterações de arquivos em um projeto.

GitHub é uma plataforma online que hospeda repositórios Git e facilita o compartilhamento e a colaboração entre desenvolvedores.

Fluxo do Git/Github

Comandos Git Comuns

git init → Inicializa um repositório vazio

git clone → Clona repositório remoto

git status → Mostra estado da árvore de trabalho

git add → Adiciona mudanças ao stage

git commit -m "msg" → Registra snapshot

git push → Envia commits ao GitHub

git pull → Sincroniza e integra mudanças

git branch → Lista ou cria branches

git merge → Mescla branches

git log → Histórico de commits

Fluxo Básico do Git/GitHub

  1. Crie um repositório no GitHub.
  2. Clone o repositório localmente com git clone [URL].
  3. Faça alterações nos arquivos do projeto.
  4. Use git add [arquivo] para preparar as mudanças.
  5. Registre as mudanças com git commit -m "mensagem".
  6. Envie as mudanças ao GitHub com git push.
  7. Para atualizar seu repositório local, use git pull.

Este fluxo básico ajuda a manter o controle de versão e facilita a colaboração em projetos.
